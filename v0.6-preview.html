📋 v0.6.0 Changelog

v0.6.0
New Features
✅ Personal Scores
Star rating (1-10) per anime. Stored in Room, synced to AniList. ScorePickerDialog with tappable stars.
✅ Real AniList Sync
Replaced stubbed saveEntry with live HTTP POST using kotlinx.serialization + HttpURLConnection. Maps status→MediaListStatus, pushes scores & progress.
✅ Airing Screen
Full rewrite — fetches 7-day schedule from AniList, grouped by day with live countdown timers, tappable cards.
✅ Enhanced Profile
Added: days watched, personal mean score, genre distribution (top 8), format breakdown, year distribution.
✅ Tappable Relations
Related anime cards in DetailScreen now navigate to their detail pages.
✅ Viewer Expansion
AnimeViewer now carries avatar URLs + AniList statistics from GetViewer.
Bug Fixes
🐛 Fixed: statusColor collision
"dropped" and "watching" both showed as Accent (pink). Now dropped=crimson, watching=rose, paused=amber, repeating=purple. Centralized all status colors in Color.kt.
🐛 Fixed: low-contrast paused pill
PAUSED pill was dark gray TextMuted → now warm amber (0xFFf0a040).
🐛 Fixed: personalScore compile error
Missing default value broke all .copy() calls. Added = null default.
Architecture Changes
• AnimeEntry: new personalScore: Int? = null field
• AnimeDatabase: migrated to v2 with fallbackToDestructiveMigration
• AniListClient.saveEntry: real HTTP POST replaces Apollo codegen stub
• AniListClient.getAiringSchedule: new method for airing data
• Color.kt: added 6 status colors + centralized statusColor() function
• AnimeViewer: expanded with avatar + statistics fields
11 files changed across the codebase
